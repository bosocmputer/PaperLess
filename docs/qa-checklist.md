## Runbook — QA 3 ข้อหลัก (end-to-end, ทำครั้งเดียวครอบทั้ง 3)
> ข้อ 2 ต้องมี workflow ที่ตีกรอบไว้ + เอกสารที่เซ็นจนจบ จึงทำเป็น flow ใหม่ (`TEST1`) เตรียมไฟล์ PDF ตัวอย่าง 1 ไฟล์

**A. ตั้ง workflow + ตีกรอบ (admin, บนคอม)**
1. login `admin` → **ตั้งค่า Workflow** → **+ สร้าง Workflow** → รหัส `TEST1` + ชื่อ → สร้างฉบับร่าง
2. **+ เพิ่มขั้นตอน** → `MAKER` / "ผู้จัดทำ" / เงื่อนไข **คนใดคนหนึ่งเซ็น** / ติ๊กผู้เซ็น **maker**
3. ส่วน **ตำแหน่งลายเซ็นบนเอกสาร** → อัปโหลด PDF ตัวอย่าง → **ลากกรอบ** ช่องลายเซ็น
4. **บันทึกขั้นตอน** → **เผยแพร่** (สถานะ → ใช้งาน)  ✅ = หน้า config + ตีกรอบ ใช้ได้

**B. นำเข้าเอกสาร (admin)**
5. **เอกสาร** → **นำเข้าเอกสาร** → อัปโหลด PDF, รูปแบบ `TEST1`, เลขเอกสารใดก็ได้ → อัปโหลด

**C. เซ็นด้วยนิ้ว (maker, บนมือถือจริง — iOS Safari + Android Chrome)**
6. login `maker` → inbox → แตะเอกสาร `TEST1`
7. **ข้อ 3:** PDF แสดง/เลื่อน/ซูมได้ไหม
8. **ข้อ 1:** วาดลายเซ็นด้วยนิ้ว — เส้นลื่น? หน้าไม่เลื่อนตามนิ้ว? ปุ่มกดง่าย?
9. ยืนยันลายเซ็น → preview → ยืนยันเซ็นเอกสาร (complete ทันที)

**D. ตรวจ PDF ฉบับจริง (admin)**
10. **เอกสาร** → เปิด `TEST1` → **ดาวน์โหลด PDF ฉบับจริง**
11. **ข้อ 2 (สำคัญสุด):** ✅ ลายเซ็นปรากฏในกรอบบนหน้าเอกสารจริง · ✅ หน้า evidence: พ.ร.บ. 2544 (ไทย) อ่านออก + verification code · ⚠️ ถ้าลายเซ็นไม่ขึ้น = gofpdi อ่าน PDF นี้ไม่ได้ → แจ้งทีม (เปลี่ยน pdfcpu)

> เก็บกวาด: ลบ `TEST1` + เอกสารทดสอบหลัง QA (แจ้งทีม)

## รายละเอียดเพิ่มเติม (checklist เต็ม)

## 1. ดีไซน์ / ทั่วไป (Trust & Calm)
- [ ] ฟอนต์ไทย (Sarabun) แสดงคมทุกหน้า ไม่มีตัวกล่อง/ฟอนต์ระบบ
- [ ] หน้าจอเล็ก (iPhone SE) + หมุนแนวนอน ไม่ล้น
- [ ] สถานะ loading / ว่าง / error ชัดเจน

## 2. Login + landing ตาม role
- [ ] `admin` login → เด้งไป **/admin/dashboard** (ไม่ใช่ inbox)
- [ ] `maker` login → เด้งไป **/inbox**

## 3. การเซ็น (signer — จุดเสี่ยงสุด, ทดสอบบนมือถือ)
- [ ] `maker` เห็นเฉพาะงานของตัวเองใน inbox
- [ ] เปิดเอกสาร → ดู PDF ได้
- [ ] **วาดลายเซ็นด้วยนิ้ว** ลื่น หน้าไม่เลื่อนตามนิ้ว
- [ ] ปุ่ม "ล้าง" มี confirm · กด "ยืนยัน" → เห็น preview ก่อน submit
- [ ] submit สำเร็จ → กลับ inbox
- [ ] "ส่งคืนเอกสาร" ต้องกรอกเหตุผลก่อน
- [ ] ลองครบเงื่อนไข: 1 (คนใดคนหนึ่ง), 2 (ทุกคน — checkerA+B), 3 (ภายนอก)

## 4. ลายเซ็นบน PDF (ฟีเจอร์หลักที่ขอ)
- [ ] **Config:** admin → ตั้งค่า Workflow → เปิด/สร้าง draft → "ตำแหน่งลายเซ็น" → อัปโหลด PDF ตัวอย่าง → **ลากกรอบ**แต่ละตำแหน่ง → บันทึก → เผยแพร่
- [ ] เซ็นเอกสารจนครบทุกขั้น → ดาวน์โหลด **PDF ฉบับจริง**
- [ ] **ลายเซ็นปรากฏในกรอบที่กำหนดบนหน้าเอกสารจริง** ⚠️ จุดสำคัญ — เช็กกับ PDF จริงของลูกค้าหลายแบบ (gofpdi อาจอ่านบาง PDF ไม่ได้ → ถ้าลายเซ็นไม่ขึ้น แต่ได้หน้า evidence แทน = ต้องเปลี่ยนไป pdfcpu)
- [ ] หน้า evidence: ข้อความ พ.ร.บ. 2544 (ไทย) อ่านออก + verification code + ตารางผู้เซ็น

## 5. Workflow config editor (admin)
- [ ] สร้าง workflow ใหม่ (doc format ใหม่) → เพิ่ม 2–3 step → เลือกเงื่อนไข + ผู้เซ็น → บันทึก → เผยแพร่ → status "ใช้งาน"
- [ ] เปิด template ที่ active → **แก้ตรงไม่ได้** + มีปุ่ม "โคลนเพื่อแก้ไข"
- [ ] เผยแพร่ template ที่ไม่มี step → ขึ้น error "ไม่มีขั้นตอน"

## 6. จัดการผู้ใช้ (system_admin)
- [ ] เห็นเมนู "ผู้ใช้" (เฉพาะ system_admin)
- [ ] เพิ่มผู้ใช้ + ตั้ง role + รหัสผ่าน → login ด้วยผู้ใช้ใหม่ได้
- [ ] แก้ role / ปิดใช้งาน · ปิด/ถอด system_admin ตัวเอง → ถูกบล็อก

## 7. เอกสารแนบ (attachments)
- [ ] admin: แนบ PDF/รูป → เห็นในรายการ → กด "ดู" เปิดไฟล์ได้ → "ลบ" ได้
- [ ] signer: เห็นไฟล์แนบ + กดดูได้ (แต่ลบไม่ได้)

## 8. Dashboard (admin)
- [ ] ไทล์ รอเซ็น/เสร็จสิ้น/ส่งคืน/ซิงก์ล้มเหลว ตรงกับความจริง
- [ ] แยกตามสถานะ + รูปแบบเอกสารถูกต้อง

## 9. External signer (ลูกค้าภายนอก)
- [ ] admin → เอกสาร → เชิญผู้เซ็นภายนอก → คัดลอกลิงก์
- [ ] เปิดลิงก์ (เครื่องอื่น/incognito) → เห็นเอกสาร → ติ๊ก consent → เซ็น → สำเร็จ
- [ ] เปิดลิงก์เดิมซ้ำ → แจ้งว่าใช้ไปแล้ว / หมดอายุ
